# 服务器交易市场插件

服务器交易市场是面向智简魔方财务系统的客户区交易插件，用于让用户将自己账户下符合条件的服务器发布到市场出售，并通过账户余额完成购买、成交入账和服务自动过户。插件内置独立 PHP 管理后台，支持挂牌审核、上下架管理、成交记录、市场设置、审计日志和历史成交价格图表。

插件以绿色为主视觉，前台与后台均做了响应式适配。购买市场在桌面端按 3 列展示，每页 15 个商品；平板与手机会自动降列。用户上架服务器较多时，上架页提供搜索、只看可出售、固定高度滚动列表和稳定选中态，避免产品过多导致页面布局异常。

## 主要功能

- 客户交易市场：展示正在出售的服务器，支持关键词、产品、最低价、最高价筛选。
- 商品详情页：展示服务器产品、服务状态、付款周期、IP 状态、到期时间、剩余时间、卖家说明和购买确认。
- 历史成交价格：按同产品统计成交均价、最低价、最高价、最近成交价，并展示 UI 图表走势。
- 用户发布出售：用户选择自己名下服务器，填写价格，选择 IP 是否正常，系统自动填入商品标题、商品详情和剩余时间。
- 隐私保护：公开市场不展示真实域名/IP；卖家说明会自动隐藏 IP 文本，建议卖家不要填写 IP 等敏感信息。
- 余额交易：买家余额扣款，卖家扣除手续费后入账，同时写入财务流水。
- 自动过户：成交后自动将服务器归属转移给买家，并保留首次订购价格、续费价格、周期等关键计费字段。
- 交易冷却：后台可配置成交后的冷却时间，冷却期内同一台服务器不能再次上架。
- 独立后台：提供挂牌审核、拒绝、上架、下架、成交记录、市场设置和审计日志管理。
- 日志管理：支持删除单条审计日志和按筛选条件清理日志；删除日志不会删除挂牌和成交数据。
- 重装恢复：插件卸载默认保留数据，重新安装或更新时会自动修复字段、索引、异常状态和重复活跃挂牌，降低数据库错乱风险。

## 安装方式

1. 将 `server_market` 目录上传到：

```text
public/plugins/addons/server_market
```

2. 在系统插件管理中安装并启用“服务器交易市场”。

3. 进入插件后台的“市场设置”，按业务规则配置手续费、价格限制、挂牌有效期、交易冷却时间和允许出售的服务状态。

4. 在客户区访问插件入口，用户即可浏览市场、发布出售和查看自己的交易记录。

## 后台入口

插件启用后，可从插件管理后台菜单进入以下页面：

- 服务器交易市场：挂牌管理、审核、上架、下架。
- 成交记录：查看交易编号、买家、卖家、成交金额、手续费和入账金额。
- 市场设置：配置市场开关、自动审核、手续费、价格限制、挂牌有效期、交易冷却时间、允许出售状态和购买提示。
- 审计日志：查看关键操作记录，并支持删除单条或当前筛选日志。

## 后台配置说明

### 市场开关

关闭后，用户不能发布和购买交易市场商品。后台仍可查看历史数据。

### 自动审核

- 关闭：用户提交挂牌后进入“待审核”，需要管理员审核通过后上架。
- 开启：用户提交挂牌后直接进入“出售中”。

### 手续费百分比

成交后从卖家收入中扣除。例如成交价 100，手续费 5%，卖家实际入账 95。买家仍按成交价支付。

### 挂牌有效天数

挂牌超过有效期后会自动下架。填 `0` 表示长期有效。

### 交易冷却时间

单位为分钟，填 `0` 表示关闭。开启后，同一台服务器成交后需要等待冷却结束才能再次上架，防止短时间反复倒卖。

### 最低和最高出售价格

限制用户发布出售时填写的价格。最高价格填 `0` 表示不限制。

### 允许出售的服务状态

使用英文状态值配置，多个状态用英文逗号分隔，例如：

```text
Active,Suspended
```

前台会显示为中文，例如“正常”“已暂停”。

### 购买提示

展示在市场和商品详情页，用于提示买家余额扣款、自动过户、售后规则等注意事项。

## 用户出售流程

1. 用户进入“我要出售”。
2. 选择可出售服务器。
3. 系统自动填入标题、商品详情、服务状态、付款周期、到期时间和剩余时间。
4. 用户填写出售价格，选择“IP 是否正常”。
5. 用户提交后，根据后台设置进入“待审核”或直接上架。
6. 买家购买成功后，该挂牌变为“已成交”，服务器自动过户给买家。

服务器不可出售的常见原因：

- 已有未完成挂牌。
- 存在未支付账单。
- 存在未完成升降级。
- 服务状态不在后台允许范围内。
- 产品为试用、按小时、按天或后付费类型。
- 同一服务器仍处于交易冷却期。

## 用户购买流程

1. 买家进入交易市场。
2. 可按关键词、产品、价格区间筛选商品。
3. 进入商品详情页查看价格、状态、剩余时间和历史成交价格图表。
4. 勾选购买确认后提交购买。
5. 系统在事务内完成余额扣款、卖家入账、财务流水写入、成交记录写入和服务器过户。
6. 成交后系统会恢复并校验服务计费字段，避免首次订购价格、续费价格或周期被过户逻辑清零。

## 隐私与安全说明

- 前台公开市场不会展示真实域名或 IP。
- 用户填写说明时不需要写入 IP，以保护卖家和买家隐私。
- 插件会自动识别并隐藏说明中的 IPv4 和 IPv6 文本。
- 买家不能购买自己的服务器。
- 同一服务器不能同时存在多个待审核、出售中或交易锁定的挂牌。
- 所有审核、上下架、购买、成交、异常恢复等关键动作会写入审计日志。

## 数据表

插件安装时会自动创建以下数据表：

- `server_market_listing`：挂牌记录。
- `server_market_trade`：成交记录。
- `server_market_log`：审计日志。
- `server_market_setting`：插件设置。

卸载插件默认不会删除数据，避免误删历史交易和审计记录。如确实需要清理数据，应先备份数据库，再由技术人员确认后处理。

## 重装与更新恢复逻辑

插件重新安装或执行更新时，会自动执行以下修复：

- 补齐缺失字段和索引。
- 修复空挂牌编号和空交易编号。
- 修复重复挂牌编号和重复交易编号。
- 下架过期挂牌。
- 下架锁定残留但未成交的异常挂牌。
- 下架服务器归属异常的挂牌。
- 下架同一服务器重复活跃挂牌。
- 清理活跃挂牌标题和说明中的 IP 敏感文本。

这些恢复逻辑用于防止插件卸载后重新安装造成数据结构不一致或交易状态错乱。

## 审计日志说明

审计日志用于记录市场关键动作，包括创建挂牌、审核、拒绝、上架、下架、取消、购买成交、计费字段恢复和异常信息。

后台可删除单条日志，也可按挂牌 ID、Host ID 或动作筛选后批量删除。删除审计日志只影响日志表，不会删除挂牌记录、成交记录或财务流水。

## UI 适配说明

- 市场列表：桌面端 3 列，每页 15 个商品；中等屏幕 2 列；手机 1 列。
- 商品筛选：产品过多时优先展示在售数量较多的产品，仍可通过关键词搜索其他产品。
- 上架页面：服务器过多时使用滚动列表，支持搜索和只看可出售。
- 管理后台：表格在移动端自动转换为卡片式布局，避免横向溢出。
- 选中态：服务器卡片选中时使用内部描边和绿色背景，不会被滚动容器裁切。

## 常见问题

### 为什么删除审计日志后挂牌还在？

审计日志只是操作记录。删除日志不会删除市场挂牌、成交记录或财务流水，这是为了避免管理员误删业务数据。

### 为什么服务器不能上架？

请检查服务器是否存在未支付账单、未完成升降级、未完成挂牌、服务状态不允许出售，或是否仍处于交易冷却期。

### 为什么公开市场看不到 IP？

这是隐私保护设计。公开市场只展示“已隐藏”，用户不需要填写 IP。

### 为什么成交后仍保留续费价格和周期？

插件在成交过户前会保存服务计费字段快照，过户后自动恢复并校验首次订购价格、续费价格、付款周期等字段，避免财务数据被置零。
